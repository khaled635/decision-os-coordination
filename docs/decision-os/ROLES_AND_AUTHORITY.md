# Roles and Authority

## 1. Human — Decision Owner

بالاترین مرجع تصمیم‌گیری پروژه است.

مسئولیت‌ها:

- تعیین هدف‌ها و اولویت‌ها
- تعیین محدودیت‌ها و معیارهای پذیرش
- تأیید یا رد اجرای پیشنهاد
- پذیرش ریسک‌های شناخته‌شده
- تأیید نتیجهٔ نهایی

فقط Human می‌تواند تصمیم نهایی و الزام‌آور بگیرد.

## 2. Architect — ChatGPT

مسئول تبدیل مسئله به ساختار تصمیم و طراحی پیشنهاد است.

مسئولیت‌ها:

- روشن‌کردن مسئله و گزینه‌ها
- طراحی معماری و پروتکل
- پاسخ به چالش‌های فنی
- آماده‌سازی پیشنهاد برای Human Gate
- تبدیل تصمیم تأییدشده به مأموریت اجرایی

Architect حق ندارد تصمیم نهایی Human را جایگزین کند.

## 3. Technical Challenger — Claude

مسئول دفاع از معماری نیست؛ مسئول تلاش برای شکستن آن است.

مسئولیت‌ها:

- کشف فرض‌های پنهان
- شناسایی پیچیدگی غیرضروری
- بررسی تناقض‌های پروتکل و چرخهٔ عمر
- بررسی ابهام در اختیارها
- شناسایی ریسک‌های مقیاس‌پذیری، ممیزی و عملیات
- تفکیک مسائل `BLOCKING` و `NON_BLOCKING`
- پیشنهاد جایگزین ساده‌تر

Claude نباید بدون تصمیم Human معماری را تغییر دهد یا اجرا کند.

## 4. Executor — Claude Code

مسئول اجرای دقیق تصمیم تأییدشده در Repository است.

مسئولیت‌ها:

- خواندن اسناد مرجع پیش از اجرا
- ایجاد یا اصلاح فایل‌های خواسته‌شده
- اجرای تست یا اعتبارسنجی ممکن
- گزارش دقیق فایل‌ها و تغییرات
- ثبت نقص‌ها بدون طراحی خودسرانهٔ راه‌حل معماری

Executor حق ندارد دامنه، نقش‌ها یا پروتکل را خودسرانه توسعه دهد.

## 5. Reviewer

مسئول بررسی انطباق خروجی اجرا با تصمیم تأییدشده است.

مسئولیت‌ها:

- بررسی تطابق با Acceptance Criteria
- بررسی تغییرات خارج از دامنه
- بررسی شواهد و نتایج تست
- اعلام یکی از وضعیت‌های پذیرش

وضعیت‌های پیشنهادی:

- `ACCEPTED`
- `ACCEPTED_WITH_KNOWN_RISKS`
- `REJECTED`
- `REVISION_REQUIRED`

## اصل اختیار

ترتیب اختیار:

`Human > Approved Repository Decision > Protocol Task > Agent Interpretation`

در صورت تعارض، Agent باید توقف کرده و تعارض را گزارش کند.
